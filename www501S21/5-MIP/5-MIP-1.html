1. Gurobi Solver Installation¶

Gurobi is a commercial MIP solver that is free for students. It is able to solve MIPs that might be difficult for the open-source GLPK solver that we have been using. You will need to download a copy of the Individual Academic License of Gurobi to your local machine. You will need to create an account using your @ncsu.edu email and then access your free named-user academic license while connected to the NCSU network (you can connect to the NCSU network using its AnyConnect VPN). On Windows, Gurobi can be installed in the default c:\gurobi911 folder, and the license file guorbi.lic should be stored in this location. After registering the software with the license, you will be able to use Gurobi anywhere (you only need to be on the NCSU network for your initial download).

Once it is installed, first, just run Guriobi. You should see a window like this:

After closing the window, you can then add Gurobi and build Gurobi in the Julia REPL in order to access it in Jupyter and allow JuMP to use it as its solver. You can test your installation by running the following:

The OR Software Stack:¶

Solver Features:¶

  • Presolve: Eliminate redundant variables and constraints; e.g.,

    $2x_1 + 2x_2 \le 1, x_1, x_2 \ge 0 \mbox{ and integer} \implies x_1 = x_2 = 0$

  • Cutting planes: Keep all integer solutions and cuts off LP solutions (Gomory cut)

  • Heuristics: Find a good initial incumbent solution
  • Parallel: Use separate cores to solve nodes in B&B tree
  • Speedup: From 1990 to 2014
    • 320,200 $\times$ computer speed
    • 580,000 $\times$ algorithm improvements
      $\implies$ 10 days of 24/7 processing (1990) $\rightarrow$ 1 second (2014)

Types of Integer Constraints¶

  1. Integer variable: Variable must be integer-valued.
    Just rounding continuous variables to the nearest integer can result in infeasibility.
    Just rounding up/down variables in a min/max problem results in a poor quality solution.
    Ex: Employee staffing (Ex 3 in LP 3).
  2. Discrete choice: Internal decision variable used to indicate choice between two or more alternatives.
    Binary variables used for yes/no choice; for example, to indicate membership.
    Ex: Set covering, matching.
  3. Consistency: Internal decision variable used to enforce required relationships with other variables.
    Important simplification: only need to enforce relationships that solver would otherwise want to violate to improve the solution.
    Ex: Minimum order quantity (in Ex 1 of LP 2), fixed-charge problems, one-time setup cost, tour constraints, scheduling, cutting stock.

using JuMP, Gurobi
b = [15, 16, 11, 17, 13, 15, 19]                   # Only 10% of values used in Ex 3 in LP 3
model = Model(Gurobi.Optimizer)
@variable(model, 0 <= x[1:7] )                     # Continuous-valued variables
@objective(model, Min, sum(x[i] for i = 1:7) )
for i = 1:7
    @constraint(model, sum(x[Int(j - 7*floor((j - 1)/7))] for j = i+3:i+7) >= b[i] )
end
set_optimizer_attribute(model, "OutputFlag", 0)
